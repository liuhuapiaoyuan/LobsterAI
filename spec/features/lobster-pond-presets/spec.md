# Feature Specification: 虾池（独立预制龙虾入口）

**Feature ID**: lobster-pond-presets  
**Created**: 2026-03-31  
**Status**: Done（已实现，见下方「实现对照」）

## Problem Statement

用户希望从侧栏**一键进入「虾池」**，在主区域**只浏览官方预制龙虾（模板）**，并清楚看到每个模板是否已加入自己的智能体列表。当前「预设虾」区块嵌在「我的虾池」整页中，与自定义智能体、创建入口、设置等混在一起，预制模板的发现路径不够独立。与此同时，中文界面里「我的虾池」这一名称若再与独立菜单「虾池」并列，容易造成语义重叠，需要在信息架构上区分「预制模板馆」与「我的智能体管理」。

## User Scenarios & Testing

### Scenario 1: 从侧栏进入虾池并浏览全部预制模板

**Given** 用户已打开应用主窗口且侧栏展开  
**When** 用户点击侧栏中的「虾池」菜单项  
**Then** 主区域切换为虾池视图，展示**全部**官方预制龙虾模板（与产品内置模板清单一致）  
**And** 每个模板以卡片（或等价列表项）形式展示名称、图标、简介  
**And** 能区分「尚未添加」与「已添加」状态  

### Scenario 2: 添加未安装的预制模板

**Given** 用户在虾池视图中看到某个尚未添加的预制模板  
**When** 用户触发该模板对应的「添加」操作  
**Then** 该模板作为智能体被加入用户的智能体列表（与产品在智能体页添加预制模板的行为一致）  
**And** 界面反映为已添加状态（或等价反馈）  

### Scenario 3: 对已添加的预制模板发起对话

**Given** 用户在虾池视图中看到某个已添加的预制模板  
**When** 用户触发「去聊天」或「切换并协作」等主操作（与现有智能体页对预制模板的一致行为）  
**Then** 当前工作智能体切换为该模板对应的智能体  
**And** 用户进入协作对话界面并可立即开始对话  

### Scenario 4: 从虾池跳转到完整智能体管理（可选）

**Given** 用户正在虾池视图中  
**When** 用户选择进入「管理我的智能体」或等价入口（若产品提供）  
**Then** 主区域切换到原有的全功能智能体管理视图，可管理自定义智能体与详细设置  

### Scenario 5: 语言与无障碍

**Given** 用户界面语言为中文或英文  
**When** 用户查看侧栏与虾池页标题、说明文案  
**Then** 侧栏显示对应语言的「虾池」名称；虾池页内显示副标题说明（英文标题采用 *Lobster Pond*，避免生硬直译导致歧义）  
**And** 侧栏焦点与选中态与技能、MCP 等其它主入口表现一致  

> **说明**：「简短说明」以**页内副标题**为主；侧栏仅展示入口名称，与现有导航一致。

## Functional Requirements

### FR-1: 侧栏独立入口

- **FR-1.1**: 侧栏提供独立可聚焦的菜单项，中文标签为「虾池」，与现有「新对话」「定时任务」「技能」「MCP」等并列。
- **FR-1.2**: 选中该菜单项时，该按钮具有与其它主视图入口一致的选中视觉样式。
- **FR-1.3**: 该菜单项使用独立文案键（例如 `lobsterPond`），不与「智能体管理」共用同一标题键。

### FR-2: 虾池主区内容范围

- **FR-2.1**: 进入虾池后，主区**仅**展示官方预制龙虾模板，不将自定义智能体创建/列表作为本页主要内容。
- **FR-2.2**: 展示**全集**预制模板（与产品内置预制清单一致），而非仅展示「尚未安装」的子集。
- **FR-2.3**: 每个模板卡片展示：名称、图标、简介；并明确标识是否已加入用户智能体列表。

### FR-3: 操作与行为一致性

- **FR-3.1**: 对未添加模板：提供一键添加能力，添加后的结果与在智能体页添加预制模板一致。
- **FR-3.2**: 对已添加模板：提供进入协作对话的路径，行为与智能体页中对已启用预制模板发起对话一致（切换当前智能体并进入协作）。

### FR-4: 国际化

- **FR-4.1**: 提供中文「虾池」及页面级说明文案（例如副标题或空状态说明）。
- **FR-4.2**: 提供英文标题与说明；英文名称需表意清晰（推荐 *Lobster Pond* 或 *Preset lobsters*），避免用户误解为生物或无关含义。

### FR-5: 命名与信息架构（与现有「我的虾池」区分）

- **FR-5.1**: 侧栏在同时存在「虾池」与「智能体管理」两项时，二者中文名称不得让用户无法区分用途。
- **FR-5.2**: **默认推荐**：将原智能体入口的中文「我的虾池」调整为 **「智能体」** 或 **「龙虾管理」**（具体用词以产品定稿为准），使「虾池」专指官方预制模板馆；英文智能体入口可保持 *My Agents* 或改为 *Agent studio* 等与「虾池」英文区分。

## Success Criteria

| Criterion | Target | Measurement Method |
|-----------|--------|--------------------|
| 到达预制模板并发起添加或对话 | 从侧栏起 ≤2 次点击可完成（进入虾池 + 一次卡片操作） | 走查与用例测试 |
| 语义可区分性 | 用户能理解「虾池」与「智能体管理」的差异 | 简短内部可用性走查或问卷 |
| 预制全集可见性 | 虾池页展示的预制数量与产品内置全集一致 | 对照内置清单逐条核对 |
| 行为一致性 | 添加/切换协作结果与智能体页预制区一致 | 对比测试 |

## Non-Goals / Out of Scope

- 不在虾池页提供自定义智能体的创建、删除、完整编辑流程（仍由全功能智能体管理视图承担）。
- 不要求在虾池页内嵌完整的系统提示词、技能多选编辑器等高级设置（可跳转到既有设置路径）。
- 不要求服务端动态下发预制列表；预制仍以应用内置清单为准（除非产品另行立项）。

## Assumptions & Dependencies

- 产品继续提供内置预制龙虾清单，且与智能体存储、添加预制、切换协作等能力兼容。
- 智能体管理视图（原「我的虾池」页）继续存在，作为自定义与高级管理能力入口。
- 侧栏主导航模式与现有多主视图切换机制保持一致。
